# 小说 AI 工作台 · AI 功能详解

本文面向使用「小说 AI 工作台」写作的作者，介绍工作台里所有与 AI 相关的功能：它们能做什么、在哪里触发、会参考哪些信息、输出什么结果、结果怎么应用，以及使用时的注意事项。每一项都按照实际功能整理，方便你边写边查。

工作台的 AI 能力大致分成几类：在正文里直接动笔的「写作动作」（续写 / 改写 / 润色 / 全章 / 灵感 / 纠错）、「伏笔」与「时间线」「人物关系图」的智能同步、随时可问的「AI 对话」，以及统一调度这些行为的「提示词管理」。

---

## 前提：配置 AI 连接（API）

在使用任何 AI 功能之前，需要先配置一个可用的 AI 连接（应用本身**不含 AI 额度**，使用你自己的接口），否则 AI 操作不会有真实结果。配置一次即可长期使用。

**支持两种连接类型：**

- **官方 OpenAI 连接**：使用 OpenAI 专属接口。
- **OpenAI 兼容连接**：使用通用的 `/chat/completions` 接口，可对接 DeepSeek、通义千问、智谱、Kimi、本地大模型（如 LM Studio）等。

**第 1 步：准备一个 API**

到对应平台的控制台创建并复制一个 **API Key**，记下它的 **Base URL** 和可用 **模型名**。常见服务（地址与模型名以各家官方文档为准）：

| 服务 | Base URL（示例） | 连接方式 |
|------|------------------|----------|
| OpenAI 官方 | `https://api.openai.com/v1` | 添加 OpenAI |
| DeepSeek | `https://api.deepseek.com` | OpenAI 兼容 |
| 通义千问（阿里百炼） | `https://dashscope.aliyuncs.com/compatible-mode/v1` | OpenAI 兼容 |
| 智谱 GLM | `https://open.bigmodel.cn/api/paas/v4` | OpenAI 兼容 |
| Kimi（月之暗面） | `https://api.moonshot.cn/v1` | OpenAI 兼容 |
| 中转 / 第三方聚合 API | 由中转服务商提供（通常形如 `https://你的中转域名/v1`） | OpenAI 兼容 |

> **关于「中转 API」（第三方聚合 / 代理）**：很多服务把 OpenAI、Claude、Gemini、国产模型等用统一的「OpenAI 兼容」接口转发出来，方便国内直接访问、或集中管理多家模型。用法和其它兼容连接**完全一样**：
> 1. 选 **「添加 OpenAI 兼容连接」**；
> 2. **Base URL** 填中转服务商给你的地址（通常以 `/v1` 结尾，例如 `https://api.xxx-relay.com/v1`）；
> 3. **API Key** 填中转方发给你的密钥；
> 4. **模型名**按该中转支持的填写（如 `gpt-4o`、`claude-3-5-sonnet`、`deepseek-chat` 等）。
>
> 注意：① 请选择稳定、可信的中转服务；② 你的正文等内容会经由中转方转发，介意隐私的请谨慎选择；③ 若「支持的模型」拉取不到列表，可手动填写一个该中转确定支持的模型名。

**第 2 步：在应用里填写**

1. 打开任意一本书 → 进入写作界面 → 「设置」面板 →「AI 连接」标签页。
2. 点 **「添加 OpenAI」**（官方）或 **「添加 OpenAI 兼容连接」**（其它服务）。
3. 填写 **连接名称**、**Base URL（API 地址）**、**API Key**（只保存在本机，不会上传），然后**拉取并选择模型**。
4. 把它**设为默认连接**即可。

配好之后，下文介绍的各项功能才会调用真实模型生成内容。（安装、数据存放等见 [安装说明](安装说明.md)。）

## 一、AI 写作动作

写作动作是直接作用于正文的 AI 操作，集中在 **AI 写作面板**。其中续写 / 改写 / 润色 / 全章 / 灵感都支持在按钮旁输入一段「补充提示」，告诉 AI 你这一次想要的方向。生成结果会进入右侧「AI 结果」面板，由你决定是否应用（灵感与纠错除外，见下文）。

### 1. 续写

- **它做什么**：根据光标锚点或正文末尾，生成接下来的内容。可以续写到光标处，也可以追加到正文末尾。
- **在哪里触发**：AI 写作面板的「续写」按钮，支持输入补充提示。
  - 补充提示 placeholder：「可选：写下接下来的发展方向、冲突、情绪或要出现的信息。」
- **会参考哪些信息**：前 5 章时间线、未回收伏笔、人物关系、资料库引用、当前章节正文；如果设置了光标锚点，则主要读取锚点前文。
- **输出什么**：续写内容文本。
- **结果怎么应用**：如果有光标锚点，插入到锚点处；否则追加到正文末尾。（结果面板按钮文字为「插入锚点／追加」。）

### 2. 改写

- **它做什么**：改写选中文本或整章正文，可以改变风格、节奏、视角或重点。
- **在哪里触发**：AI 写作面板的「改写」按钮，支持输入补充提示。
  - 补充提示 placeholder：「可选：写下要改成什么风格、节奏、视角或重点。」
- **会参考哪些信息**：资料库引用和当前章节正文。
- **输出什么**：改写后的内容文本。
- **结果怎么应用**：如果有选中文本，替换选中部分；否则替换全文。（结果面板按钮文字为「替换正文」。）

### 3. 润色

- **它做什么**：润色选中文本或整章正文，调整细腻度、克制度、画面感等。
- **在哪里触发**：AI 写作面板的「润色」按钮，支持输入补充提示。
  - 补充提示 placeholder：「可选：写下润色方向，比如更细腻、更克制、更有画面感。」
- **会参考哪些信息**：资料库引用和当前章节正文。
- **输出什么**：润色后的内容文本。
- **结果怎么应用**：如果有选中文本，替换选中部分；否则替换全文。（结果面板按钮文字为「替换正文」。）

### 4. 全章生成

- **它做什么**：生成完整章节内容，可以根据标题、已有正文和补充提示融合并补全。
- **在哪里触发**：AI 写作面板的「全章」按钮，支持输入补充提示。
  - 补充提示 placeholder：「可选：写下本章大致内容、关键场景、人物目标或结尾钩子。」
- **会参考哪些信息**：前 5 章时间线、未回收伏笔、人物关系、资料库引用、当前章节正文。
- **输出什么**：完整章节内容文本。
- **结果怎么应用**：替换正文。（结果面板按钮文字为「替换正文」。）

### 5. 灵感

- **它做什么**：在卡文或需要灵感时，生成接下来可写的灵感方向和写法建议。
- **在哪里触发**：AI 写作面板的「灵感」按钮，支持输入补充提示。
  - 补充提示 placeholder：「可选：写下卡住的位置、想要的情绪、冲突或接下来想解决的问题。」
- **会参考哪些信息**：前 5 章时间线、未回收伏笔、人物关系、资料库引用、当前章节正文。
- **输出什么**：灵感方向和可选写法建议（通常为若干条清晰短条目，说明每条适合怎么接下去写、牵动哪个冲突/关系/伏笔/计划）。
- **结果怎么应用**：**仅供参考，不直接应用到正文**。结果面板只有「取消」按钮，没有「应用」按钮（结果面板提示文字为「仅供参考」）。

### 6. 纠错

- **它做什么**：自动识别并修正文中的错别字、漏字、多字、明显误用字和标点错误。只改文字错误，不改剧情和风格。
- **在哪里触发**：「AI 纠错」面板中的「自动纠错」按钮。
- **会参考哪些信息**：当前章节标题和正文。
- **输出什么**：修改记录列表 + 修改后的正文。
- **结果怎么应用**：**直接修正当前正文，修改自动应用、无需确认。** 会显示修改数量，如「已自动完成纠错，共 N 处」，并支持查看修改历史（最多显示前 6 处）。
- **注意**：因为是自动应用，纠错后建议复核一下修改历史，确认没有误改。

### 写作动作速查表

| 动作 | 主要上下文 | 选中文本时 | 未选中时 | 是否需手动应用 |
|------|-----------|-----------|---------|---------------|
| 续写 | 前 5 章时间线、伏笔、人物、资料库、正文（锚点前文为主） | — | 插入锚点／追加到末尾 | 是 |
| 改写 | 资料库引用、当前正文 | 替换选中部分 | 替换全文 | 是 |
| 润色 | 资料库引用、当前正文 | 替换选中部分 | 替换全文 | 是 |
| 全章 | 前 5 章时间线、伏笔、人物、资料库、正文 | — | 替换正文 | 是 |
| 灵感 | 前 5 章时间线、伏笔、人物、资料库、正文 | — | 仅参考，不写入 | 否（无「应用」按钮） |
| 纠错 | 当前章节标题、正文 | — | 直接修正正文 | 否（自动应用） |

### AI 结果面板

写作动作生成的内容会显示在右侧检查器面板的「AI 结果」部分，方便你查看后再决定是否应用。

- **显示内容**：当前处理对象（选中文本／灵感参考／续写锚点／全文）、AI 生成的内容、本次使用的 AI 模型信息。展示文字为「当前处理：[对象类型]」。
- **操作**：提供「应用」和「取消」按钮；**灵感类型没有「应用」按钮**。
- 当还没有生成结果时，会提示：「选择引用资料后，点击上方 AI 操作生成结果。」

### AI 写作引用面板

写作动作所参考的资料，可以在「引用」面板里手动挑选。

- **三个按钮**：「资料库」「伏笔」「计划」。
- **资料库**分为「背景」「设定」「其他」三种类型；**伏笔**仅显示「未回收伏笔」；**计划**显示未完成创作计划。
- 点击按钮展开菜单选择条目加入引用列表，点击引用条目可查看详情，也支持删除引用。
- AI 写作操作会自动把这些引用作为上下文读取。无引用时提示「选择引用资料后，点击上方 AI 操作生成结果。」

## 二、伏笔（埋坑 / 填坑）

工作台用「坑（pit）/ 伏笔（foreshadow）」体系来管理故事线索，分为三个阶段：**坑创建 → 计划回应 → 回应验收 + 埋坑确认**。伏笔既可以手动创建，也可以由 AI 在时间线生成时自动产生。

### 1. 伏笔库管理

- **它做什么**：手动或由 AI 生成伏笔记录，管理伏笔的埋设、回收和进展状态。
- **在哪里触发**：「伏笔」面板的「新建伏笔」按钮，或在 AI 时间线生成时自动创建。
- **会参考/记录哪些信息**：伏笔内容（content）、补充说明（note）、埋设章节、回收章节、回收方式说明。
- **输出/界面**：伏笔列表和详情面板，包含伏笔内容、状态、回收情况、响应次数。
- **使用方式**：支持编辑内容和说明，记录埋设／回收章节，支持按状态筛选（全部／未回收／推进中／已回收）。
- **注意**：伏笔状态分为「已回收 / 推进中 / 未回收」；每条会显示创建方法（AI 生成 / 手动创建）；响应记录面板会显示「回收：[payoff]」信息。

### 2. 埋坑候选确认（伏笔候选确认）

- **它做什么**：在时间线 AI 更新时，提供未回收伏笔候选列表，自动匹配和同步伏笔库记录。
- **在哪里触发**：AI 时间线更新流程中自动处理（无需单独点按钮）。
- **会参考哪些信息**：前文未回收伏笔列表、当前章节时间线生成的伏笔动作。
- **结果怎么应用**：AI 返回伏笔数据时，自动通过 `pitId` 字段匹配现有伏笔库记录；新伏笔直接插入；既有伏笔同步响应记录。
- **输出**：同步摘要中包含「同步 X 条伏笔库记录」。
- **相关验收能力**：
  - **分析埋坑确认候选**：根据当前章节正文判断现有坑候选是否成立，并补充正文中新出现的有效新坑候选。
  - **生成填坑总结候选**：基于当前章节正文和计划回应的坑列表，总结填坑结果（none / partial / clear / resolved）和说明。
  - **生成本章伏笔候选**：基于当前章节正文、目标、钩子提炼 2-4 条可作为后续伏笔／未解线索的候选，每行一条。

## 三、时间线 AI 更新

时间线用来记录每章的「时间、剧情、人物状态、伏笔动作」，AI 可以一次性帮你整理这些信息。

- **它做什么**：生成章节时间线信息，包括具体时间、章节总结、剧情事件、人物状态、伏笔动作。采用两层提示：**先抽取具体时间，再生成完整时间线**。
- **在哪里触发**：时间线面板的「AI 更新章节时间线」按钮。
- **会参考哪些信息**：当前章节正文、前 5-10 章时间线、未回收伏笔、人物关系、资料库内容。
- **结果怎么应用**：自动解析 AI 返回的 JSON，保存到对应时间线字段（具体时间、章节总结、事件、人物状态、伏笔），并把伏笔同步到伏笔库。
- **输出**：更新摘要，如「已更新 X 条具体时间、Y 条章节总结、Z 条剧情、W 条人物状态、V 条伏笔，同步 U 条伏笔库记录。」
- **两个阶段对应的提示词**：
  - **第一阶段「具体时间提示词」**：从正文抽取故事时间，判断穿越前／穿越后／回忆／倒叙／插叙或正线位置，并优先标记明确写出的年代、昼夜、年龄或穿越节点。只根据正文和前文判断，不用资料库补写、不猜测未来。
  - **第二阶段「时间线提示词」**：参考前 10 章时间线，一次性生成章节总结、剧情事件、人物状态和伏笔动作；章节总结同步到规划，伏笔动作同步到伏笔库。新埋设伏笔最多 3 条。
- **注意**：失败时会显示 AI 错误信息。

## 四、人物关系图 AI 更新

- **它做什么**：生成当前章节涉及的人物节点和关系。输出的 JSON 包含人物信息（name / role / summary）和关系（from / to / label / summary）。
- **在哪里触发**：人物关系面板的「AI 更新章节关系图」按钮（有两个位置：「全部」视图和「当前章节关系」视图）。
- **会参考哪些信息**：当前章节正文、章节涉及的人物列表、前文人物关系记忆（既有关系摘要）。
- **结果怎么应用**：解析 JSON 生成章节关系图的节点和边，匹配资料库人物并自动同步。一次性更新当前章节涉及的所有人物，而不只是选中的人物。
- **输出**：生成的人物节点和关系列表，以及同步摘要，如「已生成 X 个当前章节人物节点和 Y 条关系，同步 Z 个资料库人物」。
- **注意**：有两个模式——library（资料库视图）和当前章节关系视图。若匹配失败会提示「AI 没有匹配到当前章节人物，请检查返回的人物名。」

## 五、AI 一键总结

如果你想一次性把整章的信息都同步好，可以用「AI 一键总结」。它会把时间线、计划、关系图、人物库串起来更新。

- **它做什么**：串行同步当前章节的多个信息：**更新时间线 → 判断创作计划完成状态 → 更新人物关系图 → 更新本章出现且资料库已存在的人物信息**。
- **在哪里触发**：「AI 一键总结」面板中的「更新时间线 / 计划 / 人物」按钮。
- **会参考哪些信息**：当前章节正文、前 10 章时间线、创作计划、人物关系、资料库人物信息。
- **结果怎么应用**：自动处理多个同步操作，结果直接保存到对应系统（时间线、计划、关系图、人物库）；完成后会在章节字段中标记本章「已完成 AI 一键总结」。
- **输出**：处理状态信息，以及完成摘要，如「已完成一键总结：时间线已更新，计划完成 X/Y，关系图已同步，资料库人物更新 Z 个。」
- **注意**：
  - 计划完成判断是保守的，只有明确兑现才算完成；人物更新只更新「资料库里已存在且本章实际出现」的人物。
  - 面板会显示总结状态指示点：处理中显示「处理中」，未完成显示「本章还未完成总结」，已完成章节有「已完成 AI 一键总结」标记。

## 六、AI 对话

- **它做什么**：通用 AI 对话界面，用于剧情推演、人物对白润色、上下文问答和章节复盘。会自动带入当前作品和章节上下文，支持多轮对话。
- **在哪里触发**：「AI 对话」面板的输入框和发送按钮。
  - 输入框 placeholder：「问剧情、人物、设定，或让 AI 复盘当前章节...」
- **会参考哪些信息**：当前作品信息、当前章节标题和正文。
- **结果怎么应用**：输入问题后发送，AI 返回回复；支持多轮对话；支持在对话中选择 AI 模型。
- **输出**：对话消息列表，分别标记「你」和「AI · [供应商] · [模型]」。
- **注意**：
  - 首次进入会提示：「这是一个新的对话，从下面输入问题开始。当前作品和章节上下文会自动带入。」
  - 发送过程中显示「正在思考...」。

## 七、提示词管理

工作台把 AI 的「指令模板」开放给你自定义。所有提示词都在「设置」面板 →「提示词管理」标签页里管理；作品和章节上下文会在发送时自动追加，你只需要关注提示词本身的写法。每个提示词都支持「恢复默认值」。

### 提示词总览

| 提示词 | 用途 | 用在哪个功能 |
|--------|------|-------------|
| 对话系统 Prompt | 规定 AI 的身份、语气、能力边界和回答习惯 | AI 对话 |
| 人物关系图 Prompt | 一次性更新当前章节涉及的人物关系，输出可直接保存的 JSON | 人物关系图 AI 更新 |
| 具体时间提示词 | 从正文抽取故事时间（穿越前后 / 回忆 / 倒叙 / 相对时间等） | 时间线 AI（第一阶段） |
| 时间线提示词 | 参考前 10 章时间线，生成章节总结、剧情、人物状态、伏笔动作 | 时间线 AI（第二阶段） |
| AI 一键总结提示词 | 控制时间线 → 计划 → 关系图 → 人物的串行同步流程 | AI 一键总结 |
| AI 纠错提示词 | 识别并修正错别字、漏字、多字、误用字和标点错误 | AI 纠错 |
| 续写提示词 | 续写正文（参考前 5 章时间线、伏笔、人物、资料库、正文） | 续写 |
| 改写提示词 | 改写正文（参考资料库引用和当前正文） | 改写 |
| 润色提示词 | 润色正文（参考资料库引用和当前正文） | 润色 |
| 全章提示词 | 生成完整章节（参考内容与续写一致） | 全章 |
| 灵感提示词 | 卡文时生成灵感方向，只给方向不写正文 | 灵感 |

### 各提示词说明（界面文案）

- **对话系统 Prompt**：用于规定 AI 的身份、语气、能力边界和回答习惯，下方会自动附加当前作品/章节上下文。默认角色是小说创作助手，优先帮助推进剧情、分析人物、检查设定、润色对白、复盘结构；默认中文回答，保持既有语气，避免覆盖未确认的设定。
- **人物关系图 Prompt**：用于一次性更新当前章节涉及的所有人物关系，并输出可直接保存的 JSON（不输出解释 / 标题 / Markdown）。
- **具体时间提示词**：先参考前 5 章时间线抽取故事时间，包括穿越前后、回忆、倒叙和相对时间；正文明确写出年代、昼夜、年龄或穿越节点时也要优先标记。
- **时间线提示词**：参考前 10 章时间线，一次性生成章节总结、剧情事件、人物状态和伏笔动作；章节总结同步到规划，伏笔动作同步到伏笔库（新埋设伏笔最多 3 条）。
- **AI 一键总结提示词**：串行同步当前章节——先更新时间线，再判断创作计划完成状态，更新人物关系图，并只更新本章实际出现且资料库里已存在的人物稳定信息（计划判断保守）。
- **AI 纠错提示词**：识别并自动修正错别字、漏字、多字、明显误用字和标点错误；只改文字错误，不改剧情和风格，拿不准的词不改。
- **续写提示词**：用于续写正文，包含光标锚点处理逻辑；参考前 5 章时间线、未回收伏笔、人物关系、资料库引用和当前章节正文。
- **改写提示词**：用于改写正文（参考资料库引用和当前正文），保留核心剧情与人物关系，不新增未经确认的设定。
- **润色提示词**：用于润色正文（参考资料库引用和当前正文），不做大幅扩写、不改变人物立场。
- **全章提示词**：用于生成完整章节，参考内容与续写一致（前 5 章时间线、未回收伏笔、人物关系、资料库引用和当前章节正文）。
- **灵感提示词**：卡文时生成接下来可写的灵感，参考内容与全章一致，但只输出方向、冲突、伏笔和写法建议，不直接写正文。

> 提醒：自定义的提示词会保存在你的用户数据目录中，覆盖默认值；如果改坏了，随时可用「恢复默认值」还原。

## 排错与小贴士

### AI 无响应或报错时，先检查这些

1. **是否配置了 AI 连接**：到「设置」→「AI 连接」确认已新建连接、填好 API 地址与 API Key，并设为默认连接。没有有效连接时，AI 操作不会返回真实结果。
2. **模型与地址是否正确**：官方 OpenAI 连接和「OpenAI 兼容」连接使用的接口不同；兼容连接的地址需指向支持 `/chat/completions` 的服务。可在连接设置里用模型列表功能确认能否拉取到模型。
3. **看错误提示**：
   - 时间线、关系图等操作失败时会直接显示 AI 错误信息，按提示排查即可。
   - 关系图若提示「AI 没有匹配到当前章节人物，请检查返回的人物名」，多半是人物名没对上，可检查章节关联的人物和资料库人物。
   - 若提示 `AI_OUTPUT_INVALID`，表示模型这次返回了空内容，重试一次或补充提示通常能解决。
4. **正文/上下文是否足够**：很多功能依赖当前章节正文、关联人物、关联设定等。如果本章内容太少，AI 可参考的信息就少，结果质量会下降。

### 「选中文本」与「整章」的区别

- **改写 / 润色**：如果你在正文里**选中了一段文字**，AI 只处理并替换这段选中的内容；**没有选中**时，会处理并替换整章正文。
- **续写**：如果设置了**光标锚点**，续写会以锚点前文为主，并把结果插入锚点处；没有锚点时则追加到正文末尾。
- 想精修某一段时，先选中再操作；想整体改动时，取消选中再操作。

### AI 生成的内容都需要人工确认

- 续写 / 改写 / 润色 / 全章 的结果都会先进入「AI 结果」面板，由你点「应用」才会写入正文，点「取消」即可丢弃。
- **灵感**只供参考，不会写入正文（结果面板没有「应用」按钮）。
- **纠错**是少数会自动应用的操作，修改会直接写入正文，建议用修改历史复核一遍，确认没有误改。
- AI 的判断（尤其是计划完成、伏笔回收、人物关系）可能不完全准确，应用前请按你自己的故事设定再确认一遍。
