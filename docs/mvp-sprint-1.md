# MVP Sprint 1

## 1. 当前阶段目标

在 Windows 上交付一个可运行的 Electron 桌面应用，并形成最小但真实可用的小说创作闭环：

1. 应用启动初始化（`app.init`）
2. 创建和管理小说项目
3. 创建和编辑章节
4. 维护章节规划层与正文层
5. 使用设定库（角色 / 设定条目）
6. 使用参考章节系统
7. 查看章节梗概总览
8. 统一上下文组装能力可用
9. AI 提炼梗概真实闭环可用
10. 章节工作台布局稳定
11. 章节标题 / 本章目标 AI 生成可用
12. 填坑 / 埋坑系统可用
13. 填坑 / 埋坑层交互优化完成
14. 新增坑流程智能化完成
15. 全部坑内容总览可用
16. 全部坑内容总览信息结构优化完成
17. AI 建议层最小可用

## 2. In Scope

当前阶段必须可用：

- Electron + React + TypeScript 桌面壳
- 单应用 SQLite + 多项目
- `app.init` 初始化链路
- `NovelProject` 基础 CRUD
- `Chapter` 基础 CRUD
- `StoryPit` 最小可用
- 章节工作台
  - 本章关联角色 / 设定
  - 历史章节引用
  - 填坑 / 埋坑层
  - 章节规划层
  - AI 参考层
  - 正文层
- `Character` / `LoreEntry` 最小 CRUD
- `ContextAssembler + PromptBuilder`
- `ai.extractOutline`
- `ai.generateChapterTitle`
- `ai.generateChapterGoal`
- AI 建议面板最小工作流
- 章节梗概总览（直接读取 `Chapter.outline_user`）
- 全部坑内容总览（直接读取 `StoryPit`）

## 3. 当前阶段新增重点

### 3.1 工作台布局语义稳定

需要支持：
- 保存状态显示在章节工作台标题行右侧
- 章节标题并入章节规划层
- 章节规划层位于填坑 / 埋坑层下方、AI 参考层上方

### 3.2 参考章节系统可用

需要支持：
- 当前章节展示已选参考章节
- 手动选择历史章节作为参考
- 自动推荐最近的前文章节作为参考
- 自动与手动引用共存
- AI 参考层纳入参考章节信息
- 参考章节内容优先读取 `outline_user`

### 3.3 章节梗概总览可用

需要支持：
- 在项目内集中查看全部章节的 `outline_user`
- 按章节顺序浏览
- 看到章节序号、章节标题、本章梗概、更新时间
- 某章梗概更新后，总览在当前会话中即时反映

### 3.4 统一上下文组装能力可用

需要支持：
- AI 调用前统一查询项目、章节、关联角色 / 设定、参考章节、填坑 / 埋坑
- 统一输出结构化 `ChapterAiContext`
- 统一通过 `PromptBuilder` 构造模型输入
- renderer 不自行拼 prompt

### 3.5 标题 / 目标 / 梗概 AI 候选闭环可用

需要支持：
- 从当前章节触发 AI 生成标题 / 目标 / 梗概
- 返回候选而不是直接覆盖
- 用户确认后才写回对应字段

### 3.6 填坑 / 埋坑系统可用

需要支持：
- 当前章节可选择前文已有坑进行填坑
- 不能填未来章节产生的坑
- 当前章节可新增本章埋坑
- 可通过 AI 获取新增坑候选
- 项目级查看全部坑内容

### 3.7 填坑 / 埋坑层交互优化完成

需要支持：
- 填坑 / 埋坑层默认显示缩略内容卡片
- 点击“...”后才展开查看 / 编辑完整坑内容
- 列表层不再常驻大文本编辑区
- 列表层不再堆叠保存 / 取消按钮

### 3.8 新增坑流程智能化完成

需要支持：
- 已删除“本章是否埋坑”总开关
- 当前章是否埋坑由是否实际新增坑自然体现
- 点击“新增坑”后可看到 AI 推荐坑候选
- 用户也可以不选 AI 候选，直接手动创建
- UI 中不再强调“AI 生成 / 手动编辑”的坑内容标签

### 3.9 全部坑内容总览信息结构优化完成

需要支持：
- 总览拆分为“章节坑”和“作者手动设定坑”
- 章节坑按章节分组展示
- 默认缩略显示
- 展开后查看 / 编辑详情

## 4. 明确不做

当前阶段不做：
- 章节记忆卡 / 摘要卡新体系
- embedding / 复杂语义检索推荐
- issue tracker 式任务系统
- 一致性检查引擎
- 导入导出
- 云同步
- 富文本编辑器
- Foreshadowing / TimelineEvent 完整模块

## 5. 验收标准（DoD）

### 基础能力

- 应用启动后 `app.init` 成功
- 项目与章节可创建、保存、重启恢复
- 设定库可增删改查
- 自动保存可工作

### 工作台布局

1. 保存状态显示在章节工作台标题行右侧
2. 章节标题已并入章节规划层
3. 章节规划层位于填坑 / 埋坑层下方、AI 参考层上方

### 参考章节系统

1. 当前章节可展示已选参考章节
2. 可手动选择历史章节作为参考，且只能选择当前章之前的章节
3. 不允许引用当前章自己
4. 不允许引用后续章节
5. 可自动推荐最近的前文章节
6. 自动推荐不会直接覆盖已有 `manual / pinned` 引用
7. AI 参考层会自动纳入参考章节内容
8. 参考章节内容优先来自被引用章节的 `outline_user`

### 章节梗概总览

1. 项目中可集中查看所有章节的本章梗概
2. 每项至少显示章节序号、标题、本章梗概
3. 某章梗概更新后，总览可实时或即时刷新反映
4. 总览直接读取当前 `Chapter.outline_user`，不维护冗余副本

### AI 生成工作流

1. 可基于 AI 参考层内容生成章节标题候选
2. 可基于 AI 参考层内容生成本章目标候选
3. 可从当前章节触发 AI 提炼梗概
4. 所有候选内容都需要确认后才能应用
5. 未确认前不得直接覆盖字段
6. 确认应用后，字段成功更新
7. 更新后 AI 参考层同步刷新
8. 更新后章节梗概总览同步刷新

### 填坑 / 埋坑系统

1. 填坑 / 埋坑层默认显示缩略内容
2. 点击“...”后才展开查看 / 编辑完整坑内容
3. 列表层不再常驻大文本编辑区
4. 列表层不再堆叠保存 / 取消按钮
5. 填坑内容可编辑，且修改后总览同步更新
6. 已删除“本章是否埋坑”总开关
7. 当前章是否埋坑由是否实际新增坑自然体现
8. 点击“新增坑”后可看到 AI 推荐坑候选
9. 用户也可以不选 AI 候选，直接手动创建坑
10. UI 中不再强调“AI 生成 / 手动编辑”的坑内容标签
11. 当前章节可选择前文已有坑进行填坑
12. 不能填未来章节产生的坑
13. 项目内可查看全部坑内容总览
14. 总览拆分为“章节坑”和“作者手动设定坑”
15. 章节坑部分按章节分组展示
16. 作者可以一眼看出每章有哪些坑，以及自己手动设定了哪些坑

### AI 建议工作流

1. AI 建议仍然保持待确认工作流
2. 不自动越权修改用户内容

## 6. 当前自动推荐规则

当前阶段先使用规则式推荐：

- 参考章节：
  - 上一章优先
  - 再按最近 2 到 3 章补足
- 填坑：
  - 只能从前文已有未解决坑中选择
- 新增坑 AI 候选：
  - 以当前正文为主
  - 结合标题 / 目标 / 梗概 / AI 参考层生成 2 到 4 条候选

后续阶段再考虑更智能推荐，但当前阶段先保证稳定可用。

## 7. 建议开发顺序

1. 先稳定章节工作台结构与保存状态位置
2. 落地 `StoryPit` 数据结构与 IPC
3. 重构章节内填坑 / 埋坑层交互
4. 重构项目级“全部坑内容总览”
5. 将填坑 / 埋坑继续接入 AI 参考层
6. 扩展 `ContextAssembler + PromptBuilder`
7. 打通标题 / 目标 / 梗概 / 坑位候选的确认应用流程

## 8. 本轮补充：填坑入口与布局对齐

- “填坑”与“埋坑”都以按钮入口触发，不再让“填坑”单独保留下拉选择器。
- 作者点击“选择填坑”后，可在弹窗中选择前文旧坑，并在确认前微调坑内容。
- DoD 补充：
  1. 填坑与埋坑列表在章节工作台中视觉结构对齐
  2. 填坑通过弹窗完成“选择 -> 编辑 -> 确认”
  3. 填坑确认后，章节层、总览层、AI 参考层同步刷新

## 9. 本轮补充：AI 提炼摘要的非递归行为

- DoD 补充：
  1. `ai.extractOutline` 以当前正文为核心生成候选摘要
  2. 已应用的旧 `outline_user` 不会在下一轮摘要提炼中再次被当作主要输入回灌
  3. 连续多次点击提炼时，结果不应因“旧摘要套新摘要”而出现递归重复
  4. 用户确认前不得直接覆盖 `outline_user`

## 10. 本轮补充：无正文时的摘要生成

- DoD 补充：
  1. 当当前正文为空但 AI 参考上下文足够时，仍可生成候选摘要
  2. 该候选摘要可以利用当前章节规划、关联设定、参考章节和坑位信息
  3. 该任务仍不得把当前 `outline_user` 本身重新作为输入回灌
